## Vision

### 10x Check

The 1x version (already in the design doc): a 3-minute hackathon demo of a sales copilot that finds, contacts, coaches, and follows up — Approach A. Bookable in 5h45m. Lands the technical novelty of "live copilot for solo founders" (Cresta-grade product without Cresta's enterprise contract).

**The 10x version (this plan):** the demo doesn't pitch "a sales copilot." It pitches **the salesperson's second brain — and you can't see it.** An Electron overlay invisibly present during every call, screen-capture-exempt so the prospect's recording is blank where the panel sits. Cross-conversation memory cites yesterday's call by name. Closes with "the next call will be smarter than this one." That's a takeaway-grade YC pitch — partners remember the line three days later.

### Platonic Ideal

The 12-month state: GBrain Copilot is the human-facing flagship app of GBrain, not a Chrome extension. It's the product surface where humans bring their brain to every conversation. Sales is the wedge audience because solo founders feel the pain hardest and pay fastest. The next verticals (in order): customer success, recruiting, therapy, podcasting, journalism. Each is a single config-file change away (different system prompt, different brain shape, same product). The CLI stays for power users. MCP stays for agents. Copilot is for humans.

The wedge insight nobody else has: **cross-conversation memory.** Cresta is single-call live. Gong is post-call analytics. Nobody reasons across conversations because nobody else has the brain layer. GBrain's compounding memory is the moat.

## Accepted Scope (added to this plan)

- **Electron app shell** with two windows:
  - Overlay window: transparent + always-on-top + click-through + `setContentProtection(true)` for screen-capture exemption on macOS (Zoom/OBS render the window's region as blank).
  - Dashboard window: standard, recent transcripts + recent prospects + "New outbound" button.
- **Compounding beat (P1):** a 2nd plausible transcript pre-loaded in the brain by hour 4, and the copilot's panel cites that prior call by name during the live-replay segment of the demo. Closing line: "the next call we run will be smarter than this one."
- **Demo script reframing (P2+P3):** open with result ("30 seconds ago our outbound message booked this demo"), close with flex ("we built this in 5 hours, the product pitched itself").
- **Distribution reframing (P4):** position post-hackathon roadmap as "Copilot is the human-facing flagship of GBrain — CLI for power users, MCP for agents, Copilot for humans on calls. Sales is the wedge; vertical expansion is sales → CS → recruiting → therapy."
- **Live-send beat (P7):** founder clicks SEND on the drafted follow-up during the demo, the in-audience judge gets the email mid-pitch. Send-to-team-inbox as fallback proof projected on screen.
- **Cost + consent (P8):** hour 0 checklist adds (a) confirm hackathon doesn't restrict ~$30–60 in API spend, (b) get recorded yes-on-tape from the call prospect BEFORE the call: "we're going to demo this conversation to the room — OK?"

## Risk Map (CEO-level only — engineering risks tracked in the design doc)

| Risk | Mitigation |
|---|---|
| Electron overlay screen-capture exemption doesn't work on non-Mac demo laptop | Confirm demo laptop is Mac in hour 0. Otherwise drop the "invisible to screen-share" angle and narrate the privacy story differently. |
| Pre-stage call prospect declines consent for replay | Have a backup prospect lined up (a teammate playing the role, disclosed honestly on stage). |
| Email send-live fails (deliverability, judge's phone dead) | Send-to-team-inbox fallback projected on screen as proof; narration adapts seamlessly. |
| Cross-conversation memory citation looks faked | The 2nd transcript MUST be real (or a plausible synthesized prior call), AND the brain MUST actually search/find it (not hardcoded). Demonstrable by inspecting the brain page in the dashboard if challenged. |
| 2-engineer team eats Electron budget, falls behind on brain integration | Drop ZeroEntropy (already cut). If Electron drags past hour 4, swap overlay window for a simple non-transparent BrowserWindow and narrate the privacy story without proving it on stage. |
| Hackathon org restricts API spend | Confirm at hour 0. If restricted, drop HOG deep-research, use HOG enrichment only (cheaper). |

## What Garry Tan Will Actually Remember

A YC partner watches ~200 hackathon demos. Three retention drivers from this plan:

1. **"The next call we run will be smarter than this one."** — compounding insight, takeaway-grade, repeatable to other partners.
2. **The invisible overlay.** — screen-capture exemption is a visual trick that lands. "I built this so my prospect can't see me being coached" is a sentence partners repeat.
3. **"We used our own product to pitch you."** — proves the product works AND demonstrates founder taste (recursive elegance, not feature checklist).

If two of these three land, this is a memorable demo. If all three land, it's a top-10-of-200 demo. The plan optimizes for all three.
