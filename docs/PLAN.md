## Mission (this slice only)

Make `FindPeoplePanel` and `FindEventsPanel` **actually use the brain**.

Today: both panels call `window.pmf.hog.deepResearch()` with a static criteria string and render the raw HOG response. There is no gbrain pull. There is no ZeroEntropy rerank. The "Arvya wedge" hard-coded into `DEFAULT_CRITERIA` is a string the founder typed once at hour 0, frozen in time.

Tomorrow: every sweep starts with two parallel `gbrain.search` calls. Pain context flows into the HOG prompt **and** becomes the ZeroEntropy rerank query. Named accounts/people the brain already knows flow into a dedup list so we don't re-surface accounts already in the queue. HOG results come back ranked by ZeroEntropy against actual buyer pain from prior demos — not against the static string. Top picks can be written back to gbrain in one click.

That is what "closed loop" means. The plumbing is built. It just isn't wired.

## 10x check

1x (today): "type into a box, HOG searches the web, results render." Generic. Could be anything.

10x (this plan): "the brain reads 15 prior demos, distills the pain, asks HOG to find more people who match THAT pain, ranks the results against THAT pain, and writes the winners back to the brain so the next sweep is sharper." That sentence is the demo. That sentence is the moat.

## Failure modes

| What breaks | Surface | Mitigation |
|---|---|---|
| gbrain server down | both panels | Yellow "brain offline" pill; HOG runs ungrounded; render still works. Logged to console with the error. |
| ZE API key missing | both panels | Yellow "ZE offline" pill; result order = HOG insertion order (the existing fallback). |
| HOG returns 0 results | both panels | Existing error path: "No results returned. HOG may need a broader or more specific prompt." Already handled. |
| Brain has 0 pages indexed (fresh install, no seed) | both panels | `gbrain.search` returns empty; provenance shows "0 gbrain chunks". HOG prompt falls back to the existing `DEFAULT_CRITERIA` string. We do NOT block the sweep. |
| gbrain returns chunks with no useful text (entity-only) | both panels | `loadIcpContext` filters to chunks where `chunk_text.length > 40`. If none survive, treat as "0 useful chunks" and degrade as above. |
| `putPage` fails on save-to-brain | both panels | Red toast with the error message; sweep results stay on screen. No data loss. |
| Renderer calls `pmf:ze:rerank` with 0 documents | main | Existing function early-returns `[]`. Safe. |
| Renderer calls `pmf:gbrain:search` while still booting (before token is minted) | main | Existing client mints lazily and caches. First call adds ~200ms. Acceptable. |

## Acceptance tests

### Functional

1. With all env vars set and gbrain running: open Dashboard → People view → header shows "grounded in N chunks · M dedup'd". Click search → results render → each card has a ZE score badge → top 5 are visibly the most pain-relevant.
2. With `GBRAIN_HTTP_URL` missing: panel still loads, shows "brain offline", HOG runs ungrounded, results still render in HOG order.
3. With `ZEROENTROPY_API_KEY` missing: panel grounds, HOG runs, results render in HOG order, "ZE offline" pill visible.
4. Click "Save top 10 to brain" → page appears in gbrain under `gtm/sweeps/YYYY-MM-DD-{slug}`. Verify with `gbrain query "what did the sweep on YYYY-MM-DD find?"` — returns the saved page.
5. Person whose name appears in the dedup list does NOT appear in the rendered results (client-side filter).
6. Same flow on Events view (events-only schema, no people half).

### Demo scenarios

**Scenario A: cold brain (0 pages indexed).** Both panels work. Provenance reads "0 gbrain chunks". HOG runs with default criteria. Yellow pill. We're honest: brain isn't seeded yet.

**Scenario B: warm brain (15 transcripts seeded).** Search runs. Provenance reads "8 gbrain chunks · 12 known accounts". HOG prompt has actual DealCloud / buyer-tracker pain quotes inlined. ZE rerank surfaces a PE VP who mentioned "Excel buyer tracker" in their bio above a generic M&A director. Demo win.

**Scenario C: brain returns a known account.** That account's people do NOT appear in the results. Provenance footer lists who got dedup'd ("hidden: 3 already in queue"). Click "show all" to override.
