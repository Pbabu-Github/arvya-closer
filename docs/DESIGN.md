## Problem Statement

Naveen and Prashanth at Arvya have run **15 prior demos. 2 progressed. 13 stalled.** The stalls cluster around four shapes: stale CRM (Outlook-native deal teams), wrong CRM assumptions (DealCloud, not Salesforce), security objections, and Excel buyer-tracker pain. They need a founder-led sales system that closes the gap between "we had a meeting" and "we have a pilot."

The hackathon target — by the demo bell at hour 5:45 — is a working **PMF acceleration loop** that:

1. Reads the 15 prior demos and extracts WHY they stalled.
2. Generates account-specific outreach grounded in those patterns + market signals.
3. Coaches Naveen and Prashanth IN REAL TIME during a live demo via a private overlay.
4. Writes everything back to GBrain so the NEXT call is smarter than the last.
5. Books real demos using the system. Whatever real N is — that's what we claim.

This is NOT an AI SDR. It is NOT a Cluely clone. It is a **closed founder-led sales learning loop** where the live overlay is one of eight actuators.

## What Makes This Cool

- **Recursive.** Arvya autopsies its own demos to find what to build. The system reveals its own thesis by running on itself.
- **The 15 demos are the moat.** No competitor has Arvya's prior-demo dataset. Demo Autopsy → PMF insights is a beat nobody else can run.
- **Hybrid surface = both stories land.** The web Command Center (PMF acceleration loop) AND the Electron overlay (Cluely-grade live moment) — Cluely doesn't have the loop, no PMF tool has the overlay.
- **Founder-approved actions only.** No hidden mass email, no fake personalization, no invented facts. Every customer-visible action gets human approval. The product earns trust by saying no to autonomy in the high-stakes paths.

## Constraints

- **5h 45m hackathon clock** at session start (now ~5h after /office-hours + CEO review + autoplan setup).
- **2 engineers (Naveen + Prashanth)**; one operates the demo at 5:45, one is on the booked-prospect call.
- **Build target**: `arvya_web/app/pmf-engine` (existing Next.js app) for the Command Center; `apps/closer-desktop/` for the Electron overlay (thin: one window, screen-share exempt).
- **Stack must use** (this is the GStack/GBrain hackathon): GStack (build/validate), GBrain (memory), ZeroEntropy (live-call relevance ranking — kept after the prior plan dropped it), The Hog (market intel adapter), Groq (Whisper batch + inference), Anthropic (subagent drafting).
- **Demo audience**: YC partners watching pitch ~87 of ~200. Not PE/IB buyers. Pitch the loop, not the vertical.
- **Booking claim**: honest N. Whatever you book today, that's the number.

## Premises (challenged via /office-hours and confirmed)

1. **The 15 prior demos are the moat, not a stretch.** ASSUMING they're accessible as text (transcripts, notes, or recordings to transcribe). Hour 0 critical-path verification: where are the 15 demos NOW? Drive paths, Gong export, raw notes? If they're not in usable form in hour 0, Demo Autopsy beat becomes a synthetic-from-summary play, which is WEAKER than real autopsy.
2. **Web Command Center first, Electron overlay second — but BOTH ship today.** Reliability path (Approach A) gives up the live-coaching moment. Cluely's defensibility is precisely the screen-share-exempt overlay. Approach B keeps both. The Electron is THIN (one window, no dashboard chrome — the dashboard lives in the web Command Center).
3. **The wedge is real and documented.** Outlook-native PE/IB deal teams with DealCloud + stale CRM + Excel buyer-tracker pain. The 5 scenario tests in the source PLAN.md prove the team has actually heard these objections.
4. **Demo audience cares about the LOOP, not the vertical.** YC partners aren't PE buyers. The PE/IB language belongs in the product, not the pitch. Pitch frame: "founder-led sales learning loop that compounds." Show the PE/IB content as proof, not as the headline.
5. **"Live coaching" must be sparse, not noisy.** One card. One action. Plan's principle #3. The screen-share-exempt overlay shows ONE move at a time — the audience sees the founder use it.
6. **Honest numbers > forward-looking numbers.** Whatever real N is booked during the hackathon is what the demo claims. "Credible path to 10" gets discounted by YC partners; "we booked 2 using this today" gets respected.

## Approaches Considered

### Approach A: Closed-Loop Demo (web-first, plan-faithful) — NOT CHOSEN
Effort: M / Risk: Med. Plan as written: web sidecar only. Heaviest investment in Command Center, Demo Autopsy, Proof Room, deterministic guidance engine. Most reliable. Loses Cluely-grade live moment. Cut for: ambition floor too low for a top-10 hackathon demo.

### Approach B: PMF Loop + Live Overlay Hybrid — **CHOSEN**
Effort: L / Risk: Med-High. Web Command Center (PMF loop story) + thin Electron overlay (live-coaching beat with `setContentProtection`). Demo split ~60% Command Center, ~30% overlay, ~10% receipt write. Captures both moats. Higher surface area; we manage by SHIPPING the Electron as one window with no chrome — all dashboard UI lives in the web app.

### Approach C: 15-Demo Autopsy Centerpiece (radically narrow) — NOT CHOSEN
Effort: S-M / Risk: Low. Cut to: Demo Autopsy of 15 demos live → wedge selection → ONE outreach → ONE call → ONE receipt. Lower risk, recursive elegance ("we autopsied our own demos"). Cut for: insufficient surface area for a competitive demo, no Cluely-grade live moment.

### Approach D: Mix (A + B + C, greedy) — NOT CHOSEN
Effort: XL. Cut for: 2 engineers in 5h can't do three full surfaces. Some beat would be screen-recorded, which judges sniff.

## Recommended Approach: B — PMF Loop + Live Overlay Hybrid

### Agent Swarm — 8 contracts (from PLAN.md, unchanged)

1. **Demo Autopsy** — transcripts in → pains/objections/personas/best-moments out.
2. **Market Signal** — Hog / web / manual / seed → prioritized `MarketSignal[]`.
3. **Account Research** — account+signal+ICP → dossier + workflow guess + ask/avoid.
4. **Relevance** — buyer context + Arvya truth + prior snippets → ZeroEntropy-ranked top-5 proof, top-5 prior snippets, top-3 objections. **THIS is where ZE is visible.** (Restored from the prior CEO review's cut — ZE earns its slot here, not in outbound.)
5. **Proof Builder** — dossier + relevance → Proof Room (workflow map, pain→capability, pilot plan, security note).
6. **Outreach** — dossier + Proof Room → email + LinkedIn + follow-up + approval checklist. **Founder approval required on first slice.**
7. **Live Coach** — transcript chunks + pre-call brief + relevance → one card: say / ask / avoid / show / close.
8. **Learning** — outcome + transcript → `LearningReceipt` → GBrain.

Data contracts: see PLAN.md lines 196-211 (`MarketSignal`), 354-373 (`LearningReceipt`), 379-433 (Core Types). Authoritative as-written; no changes.

**Hour 5:15–5:45 — Dry runs + contingency.**

- Pre-record a 30-sec screen capture of EACH beat as Plan B contingency.

// scripts/transcribe-drive-audio.ts

import { join } from 'path';
const DRIVE = `${process.env.HOME}/Desktop/arvya-brain-seed/drive`;
const OUT = `${process.env.HOME}/Desktop/arvya-brain-seed/transcripts`;
async function walk(dir: string): Promise<string[]> { /* recurse, return audio files */ }
async function transcribe(path: string) {
  const fd = new FormData();
  fd.append('file', new Blob([await Bun.file(path).arrayBuffer()]));
  fd.append('model', 'whisper-large-v3');
  fd.append('response_format', 'text');
  const r = await fetch('https://api.groq.com/openai/v1/audio/transcriptions', {
    method: 'POST',
    headers: { Authorization: `Bearer ${process.env.GROQ_API_KEY}` },

  });
  return r.text();
}
